# Emergence

---

## Architectural Position

Emergence defines:
the transition from local relational operators
to global field organisation.

Emergence is not directly programmed.

It appears through:
recursive aggregation.

---

## Core Principle

No single primitive contains:
the whole field.

Field behaviour emerges only when:
local operators interact across time.

---

## Emergence Requires

- recursion
- delay
- memory
- propagation
- bounded instability
- aggregation

Without these:
the system remains mechanical.

---

## Constraint

Emergence must never be replaced by:
semantic naming.

Naming an emergent state
is not the same
as modelling its formation.

---

## Final Statement

Emergence is recursive organisation
of relational possibility
across time.
